# HomeWork06 – Split the work (daxpy a chunk)

Estensione del codice `vectorlinearcombination_dynamic.c` (calcolo di
`D = a*X + Y` con un unico ciclo `for`) richiesta dall'homework: il lavoro
viene diviso in **chunk**, si calcolano le **somme parziali** per chunk, i
parametri vengono letti da un **file di configurazione** e i risultati
vengono salvati in un **file HDF5**.

## Struttura del progetto

```
homework06/
├── include/                  # header (.h) — interfacce dei moduli
│   ├── config_parser.h       # parser file "Chiave = Valore"
│   ├── daxpy_chunks.h        # calcolo D=a*X+Y a chunk + somme parziali
│   └── hdf5_writer.h         # salvataggio risultati in HDF5
├── src/                       # implementazione (.c)
│   ├── config_parser.c
│   ├── daxpy_chunks.c
│   ├── hdf5_writer.c
│   └── main.c                # programma principale, orchestra tutto
├── config.txt                 # file di configurazione di esempio
├── Makefile
└── README.md                  # questo file
```

Ogni modulo è separato in coppia header/implementazione, secondo il
principio di *information hiding*: `include/*.h` descrive **cosa** fa un
modulo (interfaccia, commentata in stile Doxygen), `src/*.c` descrive
**come** lo fa.

## Cosa fa ciascun modulo

### 1. `config_parser.h` / `config_parser.c`
Il compito chiede di leggere i parametri da un file di testo con sintassi
`Variabile = Valore`, usando "una libreria già esistente" (es.
[Config-Parser-C](https://github.com/welljsjs/Config-Parser-C)). Per non
dover scaricare una dipendenza esterna, è stato scritto un parser minimale
che replica lo stesso comportamento di quel tipo di librerie:

1. legge il file riga per riga;
2. ignora righe vuote e commenti (`#` o `;`);
3. cerca il carattere `=`, separa chiave e valore, elimina gli spazi
   superflui (`trim`);
4. salva le coppie in una tabella (`ConfigTable`) interrogabile con
   `config_get_double`, `config_get_long`, `config_get_int`,
   `config_get_string`, ciascuna con un valore di default se la chiave
   non è presente nel file.

### 2. `daxpy_chunks.h` / `daxpy_chunks.c`
Contiene sia la versione **originale** (`daxpy_naive`, un unico ciclo,
identica nello spirito a `VectorLinearCombination` del file di partenza)
sia la versione **a chunk** (`daxpy_chunked`), implementata seguendo lo
pseudocodice dell'homework:

```
number_of_chunks = ceil(N / chunk_size)
per ogni chunk_index da 0 a number_of_chunks-1:
    current_start = chunk_index * chunk_size
    current_end   = min(current_start + chunk_size, N)
    per ogni i da current_start a current_end-1:
        D[i] = a*X[i] + Y[i]
```

(Nel codice C gli indici sono 0-based, a differenza dello pseudocodice che
era scritto 1-based; l'homework stesso avvisa che lo pseudocodice "potrebbe
contenere errori" e va adattato con attenzione in fase di implementazione.)

`compute_partial_chunk_sums` calcola, per ciascun chunk, la somma dei
valori di `D` che gli appartengono (array `partial_chunk_sum`, punto 2
dell'homework). `sum_array` è una semplice riduzione usata sia per
sommare `partial_chunk_sum` sia per calcolare la somma "diretta" di `D`
come riferimento. `vectors_are_close` confronta due vettori componente per
componente entro una tolleranza (usata per il controllo di correttezza).

### 3. `hdf5_writer.h` / `hdf5_writer.c`
Salva su file HDF5:

- un dataset per ciascun chunk, sotto il gruppo `/chunks/` (es.
  `/chunks/chunk_0000`, `/chunks/chunk_0001`, ...);
- l'array completo delle somme parziali in `/partial_chunk_sum`;
- l'intero vettore `D` in `/D_full` (comodo per controlli/analisi senza
  dover riassemblare i chunk);
- alcuni attributi sul file (`a`, `N`, `chunk_size`, `number_of_chunks`),
  utili per ritrovare i parametri della simulazione con `h5dump` senza
  dover consultare il file di configurazione.

Richiede la libreria di sviluppo HDF5 (vedi sotto).

### 4. `main.c`
Orchestra il tutto:

1. legge il file di configurazione (default `config.txt`, oppure passato
   come primo argomento da riga di comando);
2. alloca e inizializza `X` e `Y` (tutte le componenti pari a `Xi`/`Yi`,
   come nel codice originale);
3. calcola `D` sia con `daxpy_naive` sia con `daxpy_chunked` e verifica che
   coincidano esattamente (**Check 1**, punto 1 dell'homework: *"Check
   that d is the same as the original code"*);
4. calcola `partial_chunk_sum` e confronta la somma di tali somme con la
   somma diretta di `D` (**Check 2**, punto 2 dell'homework), con una
   piccola tolleranza (`1e-9`) perché la somma a chunk e la somma diretta
   eseguono le addizioni in ordine diverso e possono differire per un
   errore di arrotondamento anche a parità di algoritmo corretto;
5. salva i risultati in HDF5 tramite `hdf5_writer`.

## Compilazione

Richiede la libreria di sviluppo di HDF5 installata sul sistema:

```bash
# Debian/Ubuntu
sudo apt install libhdf5-dev

# su un cluster HPC, tipicamente:
module load hdf5
```

Poi, dalla cartella `homework06/`:

```bash
make        # compila l'eseguibile ./homework06
make run    # compila (se serve) ed esegue con config.txt
make clean  # rimuove i file compilati e output.h5
```

Il `Makefile` cerca automaticamente i flag di HDF5 con `pkg-config`
(provando sia `hdf5` che `hdf5-serial`, nome comune su Debian/Ubuntu); se
non li trova, ricade su `-lhdf5` semplice. Se sul vostro sistema HDF5 è
installato in un percorso non standard, potete compilare a mano indicando
i percorsi, ad esempio:

```bash
gcc -Wall -O2 -Iinclude -I/percorso/hdf5/include \
    src/*.c -o homework06 -L/percorso/hdf5/lib -lhdf5 -lm
```

## Esecuzione

```bash
./homework06 config.txt
```

Esempio di output (con i valori di default di `config.txt`: `a=2.5`,
`Xi=1`, `Yi=3`, `N=100`, `chunk_size=8`):

```
--- Check 1: D a chunk vs D originale (ciclo unico) ---
OK: tutte le 100 componenti di D coincidono esattamente.

--- Check 2: somma a chunk vs somma diretta di D ---
Numero di chunk            = 13
Somma diretta di D         = 550.000000000000000
Somma delle somme parziali = 550.000000000000000
OK: le due somme coincidono entro la tolleranza (1.0e-09).
```

(Verificato eseguendo la stessa logica di `main.c` in questo ambiente,
usando uno stub per la sola parte HDF5 non disponibile qui: `config_parser.c`
e `daxpy_chunks.c` sono stati compilati e testati con successo.)

## Ispezionare il file HDF5 prodotto

Con gli strumenti a riga di comando di HDF5 (`hdf5-tools`):

```bash
h5dump -H output.h5          # solo la struttura (header)
h5dump output.h5             # struttura + contenuto completo
h5dump -a a -a N -a chunk_size output.h5   # solo gli attributi
```

## Modificare i parametri

Basta editare `config.txt` (o crearne una copia) e passarne il percorso
come argomento:

```
a = 3.0
Xi = 0.5
Yi = -1.0
N = 100000
chunk_size = 256
output_file = risultati_grande.h5
```

```bash
./homework06 config_grande.txt
```
