# Vector sum with single cycle and chunks
The goal of this homework is to extend the `vectorlinearcombination_dynamic.c` code, where the calculation of`D = a*X + Y` was performed with a single
for cycle: the linear combination computation is now divided in chunks, the partial sums are evaluated for every chunk and the parameters are read from 
a configuration file. Finally, the results are saved on a HDF5 file.

## Structure of the project
This project has a modular structure, which is the following:
1. an `/include` directory, containing the files
   - `config_parser.h`, which contains the definitions of the functions which read the input from a configuration file;
   - `daxpy_chunks.h`, where the functions which construct the `D` vector and perform the sums and the checks of correctness are defined;
   - `hdf5_writer.h`, where the function which saves the outputs in a HDF5 file is defined.
2. a `/src` directory, containing the files
   - `config_parser.c`;
   - `daxpy_chunks.c`;
   - `hdf5_writer.c`,
   where the function of the relative `.h` files are implemented.
   - `main.c`, where the main function of the code is implemented and performs the single and chunked definitions of `D`, the sums computation and the comparison
     between the outcomes.
3. a `Makefile`, which allows an optimized compilation of the program;
4. a `config.txt` file, from where the inputs to the program are read. Reading the parameters at runtime allows to avoid recompiling when changing input values, which may require
   a long time in more complicated codes.

## How does the code work
 
### 1. `config_parser.h` / `config_parser.c`
This part of the code reads the parameters from a `.txt` file written with the syntax `Key = Value` as follows:
 
1. reads the file, row by row;
2. eventually ignores comments or empty rows (`#` or `;`);
3. looks for the `=` symbol, separates key and value and eliminates blank spaces (`trim`);
4. saves the couples in a table (`ConfigTable`), which can be accessed through the functions `config_get_double`,
   `config_get_long`, `config_get_int`, `config_get_string`;
   
### 2. `daxpy_chunks.h` / `daxpy_chunks.c`
This files contain both the single cycle (original version) of the vector combination computation and the chunked 
version. The function `compute_partial_chunk_sums` evaluates, for each chunk, the sum of the `D` values that are
contained inside. `sum_array` is used both to sum `partial_chunk_sum` and evaluate the direct `D` sum, taken as 
reference. `vectors_are_close` compares component-wise two vectors, within a certain tolerance (used to check the
correctness of the chunked sum w.r.t. the single cycle.

### 3. `hdf5_writer.h` / `hdf5_writer.c`
This files save the output on a HDF5 file:

- a dataset for each chunk, under the `/chunks/` flag (e.g.
  `/chunks/chunk_0000`, `/chunks/chunk_0001`, ...);
- the full array of partial sums in `/partial_chunk_sum`;
- the full `D` vector in `/D_full`;
- some input parameters (`a`, `N`, `chunk_size`, `number_of_chunks`), which can be useful to find the simulation
  parameters with `h5dump` without checking the `config.txt` file.

### 4. `main.c`
This file accomplishes the following tasks

1. reads the `config.txt` file;
2. allocates and initializes alloca `X` e `Y`;
3. evaluates `D` with both `daxpy_single` and `daxpy_chunked` and verifies they coincide;
4. evaluates `partial_chunk_sum` and compares the sum of the partial sums with the
   direct `D` sum, with a small tolerance (`1e-9`), since they performs the operations in different
   orders and therefore can differ by a small rounding error, even if both algorithms are correct;
6. saves the result in HDF5 format with `hdf5_writer`.

## Compiling

The compilation requires the HDF5 development library to be installed. Given this prerequisite is satisfied, run

```bash
make        # compiles the executable file ./HW06 and creates the directory `/build`
make run    # compiles and executes with config.txt, 
make clean  # removes the compiled files and output.h5
```
The `Makefile` automatically looks for the HDF5 flags with `pkg-config`.

## Inspect the HDF5 file

The HDF5 tools allow to inspect the HDF5 file from command line (`hdf5-tools`):

```bash
h5dump -H output.h5          # gives only the structure of the file (header)
h5dump output.h5             # gives both the structure and the full file
h5dump -a a -a N -a chunk_size output.h5   # gives only the input parameters
```

## Modify the input parameters

The input parameters can be changed by modifying them in `config.txt`.
