# Matrix Multiplication in C
  This guide is meant to give a detailed description of how the C code `matrix_multiplicationc.c` is built and work.

## Goal of the code
  This C program have the goal of computing the product of two `N x N` matrices, according to the formula `C[i, j] = Σ_k A[i, k]*B[k, j]`, where:
- **A** and **B** are `N x N`-dimensional diagonal matrices, with the structure `A = a Id` and `B = b Id` where `Id` is the `N x N` identity matrix, `a, b` two constant scalars.
- **C** is the resulting matrix.

## Compiling the code
To compile the code and obtain the executable file, use the `gcc` compiler in the following way:

```bash
gcc matrix_multiplication.c -o nameofexecutable
```
or
```bash
gcc -pg matrix_multiplication.c -o nameofexecutable
```
The `-pg` option enable the functions profiling, providing information about how many times every function is called in the program and how time-consuming they are. This can be helpful to targeted code optimization.

## Structure of the code
- The user is asked to provide the constant values of **a** and **b**, the dimensionality **N** of the matrices and the name **fileout** of the file where the resulting matrix will be printed on from the command line, as shown in the following:
  ```bash
  ./nameofexecutable a b N fileout
  ```
- If the syntax is incorrect, the program prints:
  ```bash
    Usage: ./nameofexecutable a b N fileout
  ```
- Instructions are printed at runtime.
  
- The **A** and **B** matrices are created using dynamic allocation, through the function `CreateMatrix(N)`, as `N x N` dimensional vectors.
  
- The `MatrixProduct()` function takes as inputs the **A**, **B** matrices and the dimensionality **N** and gives as output the product matrix **C**. In this function, the loop ordering is the most intuitive one, i.e., `i-j-k`.
  
- The `CreateOutputFile()` function opens, writes and closes the file with the matrices **A**, **B** and their product **C = AB**.

-  Since memory is dynamically allocated, it is then freed using `free()`, after the code's task is accomplished.
   
## Faster check

A check that the programs successfully computed the result matrix is implemented. Instead of comparing all the **C** elements with the expected result, only the **diagonal** entries are checked against the constant `a · b`. Specifically, it is verified that the absolute value of the difference `C[i][i] - a · b` is greater than a certain threshold. If component `i*N + i` is different from the expected outcome, an error message will be printed out:
  ```bash
  Component number i*N + i is wrong.
  ```

## Benchmarking the algorithm
A goal of the code is to perform a benchmark of the MatMul algorithm, i.e., analyzing the loop structure of the matrix multiplication and optimizing the order to minimize the time expense. The matrix is built faster if the accesses to the cache are made in a sequential way, that is, avoiding jumps as much as possible. The matrices' elements are stacked sequentially for each row, while jumps are forced from column to column.

The function `BenchmarkMatMulAlgorithm()` performs the benchmark in the following steps:
1. asks as input from the user the number **M** of reiterations of the benchmark procedure;
2. initializes a two time variables with `struct timespec t1 t2`;
3. warms the cache up performing the `i-j-k` loop 3 times;
4. builds the product matrix and fill it through all the three possible loop ordering (`i-j-k`, `i-k-j`, `k-i-j`);
5. evaluates and piles up the **M** time intervals for each loop order and then computes the average;
6. produces an output file named `dati.txt`, containing four columns (**N**, **t_ijk**, **t_kij**, **t_ikj**). Everytime the user runs the executable, for different dimensionalities of the matrices, the new results are appended to this same file.

Finally, `dati.txt` is used to plot the behavior of the three loop orderings against increasing values of **N**, as shown in the figure below. The algorithm was applied to the cases with `N` running from 0 to 3000, with 10-large steps till 100, then 100-large till 1000, then 500-large till 3000.



It is clear that the `i-k-j` loop ordering is the fastest, since both `A[i, k]` and `B[k, j]` are accessed row-wise, minimizing the number of jumps in the cache memory. The `k-i-j` loop ordering is as performative as `i-k-j` for low `N`, while it becomes less fast for high dimensionality. The `i-j-k` loop ordering is the least performative because of multiple cache jumps.
