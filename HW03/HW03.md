# Matrix Multiplication in C
  This document provides a detailed description of the C code `matrix_multiplicationc.c`.

## Goal of the code
  This C program computes the product of two `N x N` matrices, according to the formula `C[i, j] = Σ_k A[i, k]*B[k, j]`, where:
- **A** and **B** are diagonal matrices of size `N x N`, defined as `A = a ⋅ I` and `B = b ⋅ I` where `I` is the `N x N` identity matrix, `a, b` two scalar values.
- **C** is the resulting matrix.

Since both matrices are diagonal, the expected result is `C = (a⋅b) ⋅ I`.

## Compiling the code
To compile the program:

```bash
gcc matrix_multiplication.c -o nameofexecutable
```
For profiling:
```bash
gcc -pg matrix_multiplication.c -o nameofexecutable
```
The `-pg` flag enables profiling by inserting instrumentation code that records how many times each function is called and how much time is spent inside it. The collected data is stored in the file `gmon.out` and can be analyzed using `gprof`.

## Structure of the code
- The program takes input parameters from the command line: **a** and **b**, the dimensionality **N** of the matrices and the name **fileout** of the file where the resulting matrix will be printed on, as shown in the following
  ```bash
  ./nameofexecutable a b N fileout
  ```
- If the syntax is incorrect, it prints:
  ```bash
    Usage: ./nameofexecutable a b N fileout
  ```
- Instructions are printed at runtime.
  
- The **A** and **B** matrices are dynamically allocated using the function `CreateMatrix(N)`, as `N x N` dimensional vectors.
  
- The matrices are initialized as diagonal, so all elements are set to zero and diagonal ones are set to `a` and `b`.
  
- The `MatrixProduct()` computes the product using the standard triple loop (`i-j-k` ordering).
  
- The `CreateOutputFile()` writes matrices **A**, **B** and their product **C = AB** to the output file.

- Allocated memory is released using `free()`, after the code's task is accomplished.
   
## Fast correctness check
Instead of checking all elements of C, only the diagonal entries are verified.

Since the expected result is `C = (a⋅b) I`
- all diagonal elements should be equal to `a · b`
- off-diagonal elements are expected to be zero, as from how they were initialized.

The program checks only that that the absolute value of the difference `C[i][i] - a · b` is greater than a certain threshold. If component `i*N + i` is different from the expected outcome, an error message will be printed out:
  ```bash
  Component number i*N + i is wrong.
  ```

This reduces the complexity of the check from `O(N^2)` to `O(N)`.

## Benchmarking the algorithm
The program benchmarks different loop orderings of matrix multiplication to determine which is the fastest.

### Key idea
Matrix elements are stored in **row-major** order, meaning that elements of the same row are contiguous in memory. Therefore:
- Accessing elements row-wise is cache-friendly
- Accessing elements column-wise causes cache misses

### Benchmark procedure
The function `BenchmarkMatMulAlgorithm()`:
1. Asks and reads as input from the user the number **M** of reiterations of the benchmark procedure
2. Uses `clock_gettime()` for high-resolution timing
3. Performs a short warm-up to stabilize cache effects
4. Executes the multiplication using three loop orderings:
   - `i-j-k`
   - `i-k-j`
   - `k-i-j`
5. Measures execution time for each ordering
6. Computes the average over **M** runs
7. Produces an output file named `dati.txt`, containing four columns (`N t_ijk t_kij t_ikj`). Everytime the user runs the executable, for different dimensionalities of the matrices, the new results are appended to this file.

Finally, `dati.txt` is used to plot the behavior of the three loop orderings against increasing values of **N**, as shown in the figures `timeintervals.png` and `timedifferences.png`.

## Results and discussion
The benchmark shows that:
- `i-k-j` is the fastest ordering
- `k-i-j` has similar performance for small `N`, but becomes slightly slower for large `N`
- `i-j-k` is the slowest

### Explanation
1. `i-k-j`:
   - Accesses both `A[i, k]` and `B[k, j]` efficiently
   - Writes to `C[i, j]` sequentially
   - Maximizes cache locality
2. `k-i-j`:
   - Reuses `A[i,k]` efficiently, but doesn't access it in the most efficient way
   - Still benefits from good memory access patterns
3. `i-j-k`:
   - Accesses `B[k, j]` with large strides
   - Causes frequent cache misses
   - Results in poor performance
### Final remark
Even though the matrices are diagonal, the full matrix multiplication is still performed. This makes the benchmark useful for studying memory access patterns, rather than exploiting the sparsity of the matrices.
