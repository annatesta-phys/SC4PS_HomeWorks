# Stability of Legendre Polynomial

The goal of this project is to compare different numerical ways of computing the Legendre polynomials, addressed as $P_l(x)$, at various values of $x$, specifically $x = 0.1,0.5,0.9,0.99$ for $l\leq 50$. It is of particular interest the study of the numerical behavior of two algorithms (a forward and a backward one) to evaluate the polynomials, to check whether they are stable and give the correct solution allowed by the numerical precision of the architecture. The routines are the following

### Direct three-term forward recurrence
Set the initial conditions $$P_0(x) = 1, \ P_1(x) = x$$ and then use the recursive formula

$$P_{\ell+1}(x) = \frac{(2\ell+1)\cdot x\cdot P_\ell(x)}{\ell+1} - \frac{\ell\,P_{\ell-1}(x)}{\ell+1}, \ \ell\geq 1.$$

This formula is standard and exact. The only sources of error lie in $x$: since it is a real generic number, it cannot be reproduced with arbitrary precision by the computer and will necessarily carry a numerical error. During a recursive routine, the function is called many times and the numerical errors can cumulate and lead the algorithm to divergence. We want to check if this algorithm is stable or not with respect to numerical errors.

### Backward recurrence experiment

Solving algebraically the forward recurrence formula for $P_{\ell-1}$, it follows

$$P_{\ell-1}(x) = \frac{2\ell+1}{\ell}\cdot x \cdot P_\ell(x) - \frac{\ell+1}{\ell} \cdot P_{\ell+1}(x).$$

This formula can also be applied recursively, proceeding backwards. Setting 

$$\tilde{P}_{\ell_\mathrm{max} + 1} = 0, \ \tilde{P}_{\ell_\mathrm{max}} =1$$

a backward propagating algorithm can be built for 

$$\tilde{P}_{\ell},$$

which can be converted easily to $P_{\ell}$:

$$P_\ell^{(\mathrm{back})}(x) = \frac{1}{\tilde{P}_0(x)} \cdot \tilde{P}_\ell(x)$$

---
The two algorithms can be then compared with a high-precision reference, which is obtained implementing an improved version of the forward recurrence, with `long double` precision. The check is done by evaluating the absolute and relative errors of both algorithms with respect to the reference, that is taken as the "exact" result.

## Structure of the code

The code is divided in multiple files, to improve its modularity and reusability. The structure of the directories and files is the following:
1. the `include/` directory only contains `legendre_utils.h`, where the global functions of the recurrence algorithms and the reference evaluation are defined;
2. the `src/` directory contains two files, i.e.
    - `legendre_utils.c`, where the functions that were defined in `legendre_utils.h` are now fully implemented;
    - `legendre.c`, where the main function of the code is defined, together with a local function whose task is to write `.csv` files (one for each value $x$) containing the $\ell$ values, for $l\in[0,50]$, the fixed $x$ values, the results for $P_\ell(x)$ for the forward, reference and backward cases, the absolute and relative errors for the forward routine and the absolute and relative errors for the backward routine;
3. the `scripts/` directory contains `plot_legendre.py`, which reads the columns of the `.csv` files and plots the absolute and relative errors of the algorithms against $\ell$ at fixed $x$, and $P_\ell(x)$ (obtained with the forward recurrence) against $\ell$, at fixed $x$;
4. the `Makefile` is used to optimize the compilation of the program. To compile the code, first run the command
  ```bash
    make run
  ```
this builds the directories `build/` and `output/`, which contains the `.csv` files, then run the command
  ```bash
    make plot-python
  ```
to generate the plots, which can be find in the newly created directory `plots/`.

If you want to delete all the outputs of the program, simply run
  ```bash
    make clean
  ```

### Stability of the three-term direct recurrence
The three-therm relation (known as Bonnet’s recurrence), that is 

$$P_{\ell+1}(x) = \frac{(2\ell+1)\cdot x\cdot P_\ell(x)}{\ell+1} - \frac{\ell\,P_{\ell-1}(x)}{\ell+1}, \ \ell\geq 1.$$
​
is a second-order linear difference equation in $\ell$, meaning that its solution space is two-dimensional, i.e., it has two linearly independent solutions. One basis vector is the ordinary Legendre polynomial $P_\ell(x)$, the other is related to the Legendre function of the second kind $Q_\ell(x)$, that for $|x| < 1$ grows geometrically with $\ell$, while $P_\ell(x)$ stays bounded by 1. Any recurrence-based algorithm is evolving a linear combination of these two solutions through this space, even when setting the initial condition to be equal to one of the two: in fact, the initial condition itself cannot be defined exactly due to finite numerical precision and will be contaminated by the other solution; which solution survives during the ODE evolution depends on whether you propagate forward or backward, and on which solution is "dominant" in that direction.

We notice from the plots that the forward absolute and relative errors are both of order $O\Bigl(10^{-15}\Bigr)$, which is compatible with the machine numerical precision. This implies that in the forward direction, for $|x| < 1$, the recurrence is stable and the algorithm does not diverge.

### Instability of the backward recurrence

Miller's trick is useful when the solution you want is the minimal one in the direction you're not propagating in. This means that you want the recessive solution, and propagating away from it (backward) suppresses the unwanted dominant solution because round-off in the dominant direction decays relative to it. That's the case for, e.g., modified spherical Bessel functions or some hypergeometric-type recursions: the wanted solution is minimal forward, so you start the recursion downward from a far-away guessed point and the error in the unwanted dominant solution gets divided out as you propagate back to small index.

### Connection to the spherical harmonics

The spherical harmonics are connected to the Legendre polynomials as follows

$$Y_{\ell 0}(\theta,\phi) = \sqrt{\frac{2\ell+1}{4\pi}} \cdot P_\ell(\cos\theta).$$

Since $\cos\theta\in[-1,1]$, the relevant regime is exactly the one we have been testing. Notice that the error of the spherical harmonics is directly related to the error of the Legendre polynomials up to a normalization factor $0.3 \leq \sqrt{\frac{2\ell+1}{4\pi}} \leq 2.8$, which is a fixed, exactly-computable multiplicative constant, which doesn't amplify or damp the relative error at all. Therefore we expect to have a convergent result for the forward propagation (error still compatible with the machine precision).
On the other hand, using the backward Miller-style recursion to build $P_\ell(\cos\theta)$ would corrupt $Y_{\ell 0}$ by relative errors of order $O(1)$ or larger (as can be seen in the plots), i.e. it would produce garbage spherical harmonics.
