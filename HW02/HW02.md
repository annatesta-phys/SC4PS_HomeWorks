# Linear combination of two vectors in C
  This guide is meant to give a detailed description of how the C codes `vectorlinearcombination_static.c` and `vectorlinearcombination_dynamic.c` are built and work.

## Goal of the code
  These C program have the goal of computing the linear combination of two N-dimensional vector, according to the formula D = a · X + Y, where:
- **a** is a scalar,
- **X** and **Y** are **N**-dimensional vectors, with the same value for all entries (**Xi**, **Yi** fixed for all i),
- **D** is the resulting vector.

## Compiling the code
To compile the codes and obtain the executable file, use the `gcc` compiler in the following way:

```bash
gcc vectorlinearcombination_typeofmemoryallocation.c -o nameofexecutable
```
The `-o` option allows to choose the name of the executable file.

## Structure of the code
- The user is asked to provide the constant values of **a**, **Xi**, **Yi** and the dimensionality **N** of the vectors from the command line, as shown in the following:
  ```bash
  ./nameofexecutable n a Xi Yi N
  ```
- If the syntax is not respected, an error will be printed out, saying `Usage: ./nameofexecutable a Xi Yi N`.
  
- Some instructions will be printed out once the program is started.
- The **X** and **Y** vectors are built using static (in `vectorlinearcombination_static.c`) or dynamic (in `vectorlinearcombination_dynamic.c`) allocation of memory, through static arrays or pointers respectively.
- In the `vectorlinearcombination_static.c` case, the resulting vector is build directly in the main function of the code.
- In the `vectorlinearcombination_dynamic.c` case instead, a function named `VectorLinearCombination(a, X, Y, N)` is called, taking as input the scalar, the two initial vectors and the dimension N and building the linear combination vector **D**, dynamically allocating the memory.
- A check that the programs successfully computed the result vector is implemented: the expected result for every entry is computed as `expected = a · Xi + Yi` and then compared to the resulting vector **D** entry values. If component `k` is different from the expected outcome, an error message will be printed out: `Component number k is wrong`.
- When memory is dynamically allocated, I made sure of freeing it through the command `free`, after the code's task is accomplished.

## Running the code for different dimensionalities
  A test can be performed using the values `N = 10, 1e6, 1e8` for the dimensionality of the vectors, checking what happens when increasingly bigger arrays are allocated. 
   In the `vectorlinearcombination_static.c` case, we get a `segmentation fault (core dumped)` error. 
## Running the code for floating numbers
  A test can be computed using the values ` a = 3`, `Xi = 0.1`, `Yi = 7.1`. The expected result for every component is `Di = 7.4` and can be checked with the C program described above.
